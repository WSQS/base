# Language

## Purpose and Positioning
这是一门为了学习编程语言实现与语言设计而逐步构建的小型解释执行语言，并且目标上将沿着可自举语言的方向逐步演化。当前阶段优先追求：概念清楚、实现可控、行为可测试，以及优先补齐那些有助于表达和解释真实代码的语言能力，在小步扩展中逐步沉淀出更稳定的语言核心结构，为未来向可自举语言继续成长打基础。

## Current Stage
当前语言已经完成最小可运行闭环，并进入“在继续扩展能力的同时，逐步显性化语言设计选择”的阶段。

当前重点是：
- 继续增强语言表达能力
- 继续整理 parser / AST / evaluator 的核心结构
- 让语言沿着可自举方向逐步演化，但不一步到位追求完整自举能力
- 优先补齐那些对“写出更完整程序”以及“解释更真实代码结构”有基础价值的语言能力，而不是追求表面功能堆积

## Program Structure
当前程序由一组按顺序执行的顶层语句组成：

```text
program = statement*
```

当前没有：
- 代码块
- 嵌套语句块
- 模块系统
- 顶层之外的独立作用域层次

## Statements
当前支持以下语句：

### 1. `let` 声明语句
语法：

```text
let name = expr;
```

含义：
在当前环境中创建一个名字绑定，并将右侧表达式的求值结果存入该名字。

例子：

```text
let x = 1 + 2;
```

### 2. `print` 语句
语法：

```text
print expr;
```

含义：
求值表达式，并将结果输出。

例子：

```text
print x;
print 1 + 2 * 3;
```

当前不支持：
- 赋值更新语句（如 `x = x + 1;`）
- 表达式语句
- 条件语句
- 循环语句
- 函数定义或返回语句

## Expressions
当前支持以下表达式形式：

### 1. 整数字面量
例如：

```text
1
42
```

### 2. 标识符引用
例如：

```text
x
foo
```

### 3. 二元表达式
当前支持：

- `+`
- `-`
- `*`
- `/`

例如：

```text
1 + 2
x * 3
```

### 4. 括号表达式
例如：

```text
(1 + 2)
(x + 3)
```

### 当前优先级与结合性
当前表达式规则为：

- `* /` 高于 `+ -`
- 括号可以改变默认分组
- 当前表达式链按左结合构造 AST

例如：

```text
1 + 2 * 3
```

按：

```text
1 + (2 * 3)
```

解析。

而：

```text
10 - 3 - 2
```

按：

```text
(10 - 3) - 2
```

解析。

当前不支持：
- 一元运算（如 `-x`）
- 比较运算（如 `<`, `>`, `==`）
- 布尔字面量与逻辑运算
- 函数调用表达式
- 赋值表达式

## Runtime Model
当前运行时模型如下：

- 程序从第一条语句开始顺序执行
- 当前只有一个全局环境
- `let` 会在该环境中创建绑定
- 标识符表达式会从该环境中读取值
- 当前表达式求值结果只有整数值
- 当前 `print` 会输出表达式结果

当前没有：
- 多层作用域
- 局部环境链
- 闭包
- 用户自定义类型
- 堆对象模型
- 异常机制

## Current Limitations
当前语言明确不支持：

- 赋值更新
- 一元运算
- 比较运算
- 布尔值
- 字符串
- 数组 / 列表 / 对象
- 条件语句
- 循环语句
- 函数定义与调用
- 嵌套作用域
- 模块系统
- 错误恢复完善的 parser

## Near-Term Evolution
近期最可能继续考虑的方向：

- 为语言加入新的小型表达式或语句能力，尤其是那些能增强程序组织能力、并帮助解释更真实代码结构的基础机制
- 在增强能力的同时继续整理 parser / evaluator 结构，让语言核心更稳、更容易继续扩展
- 优先考虑既能增强语言表达能力、又能帮助语言逐步长成未来可自举语言核心的能力
- 近期扩展更偏向“补齐真实小语言所需的基础能力”，尤其是那些能直接帮助语言承载和解释更完整代码的能力，而不是追求表面上更丰富但结构负担更大的特性

近期不会追求：

- 一步到位做成完整自举语言
- 工业级工具链
- 大而全的通用语言设计

## Design Notes
当前几个关键设计取向：

- 先用很小的语言核心建立稳定闭环，再逐步扩展
- 当前语句系统保持极小，只支持声明与输出
- 当前值模型保持极小，只支持整数
- 当前表达式系统已经开始按优先级和结合性形成更真实的结构
- 后续新增能力不只是实现练习，也应被看作语言设计选择的一部分，并优先服务于“让这门语言能够解释更真实代码”的目标
