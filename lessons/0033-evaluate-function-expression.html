Lesson 0033

求值函数表达式

函数字面量和调用已经有 AST 了。这一课只做 evaluator：让函数成为运行时值，让函数调用真正执行。

这一步为什么重要： 这是函数从静态 AST 到动态运行的最后一步。做完这一课，let add = fn(x, y) { x + y }; print add(1, 2); 就会输出 3。语言第一次拥有了用户可定义的抽象能力。

本课范围

  • 给 Value 新增函数值变体
  • eval_expr(Expr::Fn) 返回函数值（不求值 body）
  • eval_expr(Expr::Call) 执行函数调用：求值 func 和 args，创建局部作用域，求值 body
  • 新增 evaluator 测试

这一课不做：

  • 不做闭包（不捕获外部变量）
  • 不做多语句函数体
  • 不做递归
  • 不改 scanner / parser / AST

这节课的核心知识：函数值 + 调用执行

1. 函数值

函数字面量求值时，不是立刻执行 body——而是返回一个函数值，包装参数名列表和 body 表达式。

eval_expr(Expr::Fn { params, body }, env) => Value::Fn { params, body }

这和 Expr::Number 变成 Value::Integer 的逻辑是一样的——AST 节点变成运行时值。

2. 函数调用

调用发生时：

  1. 求值 func 表达式，得到 Value::Fn { params, body }
  2. 依次求值所有实参表达式
  3. 创建 局部环境：克隆当前环境，把参数名和实参值绑定进去
  4. 在局部环境中求值 body 表达式
  5. 返回 body 的求值结果
// f(1, 2) 的执行步骤
1. func = eval_expr(Ident("f"), env) => Value::Fn { params: ["x","y"], body: Binary { ... } }
2. arg1 = eval_expr(Number(1), env) => Value::Integer(1)
3. arg2 = eval_expr(Number(2), env) => Value::Integer(2)
4. local_env = env.clone() + { "x" -> 1, "y" -> 2 }
5. result = eval_expr(body, local_env) => Value::Integer(3)

局部作用域的细节

当前的全局环境是 HashMap<String, Value>。调用函数时，需要在这个基础上加一层局部绑定。

最简单的方案：复制一份环境，把参数绑定进去，传给 body 求值。

let mut local_env = env.clone();
for (param, arg) in params.iter().zip(args.iter()) {
    local_env.insert(param.clone(), arg.clone());
}
eval_expr(body, &local_env)

注意：复制环境意味着函数内部对环境的修改不会影响外部。这是有意为之的简化——真正的闭包语义后面再说。

推荐的最小实现方向

  1. 在 Value 中加入 Fn { params: Vec<String>, body: Box<Expr> }
  2. 在 eval_expr 中为 Expr::Fn 添加分支：返回 Value::Fn
  3. 在 eval_expr 中为 Expr::Call 添加分支：按上述步骤执行调用
  4. 新增 evaluator 测试，验证最小调用场景

目标输入 / 输出

输入:
let add = fn(x, y) { x + y };
print add(1, 2);

期望输出:
3
输入:
let double = fn(x) { x + x };
let result = double(5);
print result;

期望输出:
10
输入:
let constant = fn() { 42 };
print constant();

期望输出:
42
