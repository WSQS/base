Lesson 0016

让分层 parser 支持表达式链

这一课只教一件事：让你已经拆层的表达式 parser，不再只会处理单个二元拼接，而是开始稳定支持同层连续表达式。

和你的 mission 的关系： 你已经把表达式 parser 整理成了更清晰的层次，现在最自然的下一步不是再加新语法，而是让这些层真正能承受更多真实输入。这样既符合“小步扩展 + 及时整理”，也能让 parser 更像一个能继续成长的核心结构。

今天只做这一件事

本课只关注“同层连续表达式”是否能被当前分层 parser 正确处理。范围严格控制在：

  • 支持连续加减，例如 1 + 2 + 3
  • 支持连续乘除，例如 2 * 3 * 4
  • 支持混合链，例如 1 + 2 * 3 + 4
  • 仍然保留括号和已有优先级

也就是说，你要让 parser 从“只会一层拼一次”进化到“能在同一层不断继续拼接”。

这一课真正要建立的新直觉

表达式层不是“进来一次，拼一个 Binary 就结束”。更自然的思路是：

  • 先解析当前层的左边
  • 只要后面还是这一层负责的操作符，就继续往右吃
  • 每吃一个，就把结果重新变成新的左边

这意味着每一层最终都更像一个“循环拼接器”，而不是一次性的 if 分支。

推荐的最小实现方向

  1. 先让 parse_mul_div() 能连续处理多个 * /
  2. 再让 parse_add_sub() 能连续处理多个 + -
  3. 每一层都沿用“已有结果继续当左边”的思路
  4. 保持 parse_primary() 的职责不变

也就是：本课不是重命名函数，而是让这些层真正拥有“连续处理”能力。

目标：

最终输出 11

完成标准

  • 你的加减层已经能处理连续加减
  • 你的乘除层已经能处理连续乘除
  • 你的优先级和括号能力仍然保持正确

这节课故意不做什么

  • 先不引入比较运算或赋值表达式
  • 先不同时整理 scanner 或 evaluator
  • 先不追求完整通用表达式框架
  • 先不加入新的语句种类

这一课的目标只是让现有分层结构能承受“连续同层表达式”。

卡住时优先检查这 4 件事

  • 你是不是仍然在某一层只拼一个 Binary 就立刻返回？
  • 你是不是在循环处理中忘了把“新结果”重新作为左边？
  • 你是不是修好了链式表达式，却把优先级关系搞坏了？
  • 你是不是试图一次处理太多新运算类型？
本课后的观察点： 完成这一课后，你的 parser 分层将不只是“看上去更清楚”，而是真的开始拥有更像真实表达式解析器的持续处理能力。
