Lesson 0004

扩展更多单字符运算符：`-` `*` `/`

这一课只教一件事：沿用你在 `Plus` 上已经掌握的增量模式，把扫描器继续扩展到 `Minus`、`Star`、`Slash`。

和你的 mission 的关系： 你正在把一个只能处理数字和加号的最小扫描器，逐步扩展成真正能支撑表达式语言的 lexer。今天的重点不是新概念，而是通过重复同一种扩展动作，把 scanner 的可扩展性练扎实。

今天只做这一件事

在保留现有 `Number` 与 `Plus` 的基础上，继续新增这三个 token：

  • Minus 对应字符 -
  • Star 对应字符 *
  • Slash 对应字符 /

暂时不做：括号、等号、分号、标识符、关键字。

目标输入 / 输出

输入: 12 - 3 * 4 / 2
输出: Number(12)
      Minus
      Number(3)
      Star
      Number(4)
      Slash
      Number(2)

你不需要理解运算优先级。扫描器只负责把字符流切成 token 序列。

你要亲手完成的步骤

  1. 在 enum Token 里新增 Minus、Star、Slash
  2. 在扫描循环中，分别给 '-'、'*'、'/' 添加分支
  3. 沿用你在 '+' 上已经写出来的模式：先结束当前数字，再推入对应 token
  4. 补充单元测试，覆盖混合输入和无空格输入

卡住时优先检查这 3 件事

  • 新增字符分支时，是否都先结束了当前正在累计的数字？
  • 无空格输入是否也能工作？
  • 是不是某个运算符分支少写了 result.push(...)？
本课后的观察点： 这节课结束后，你很可能会明显感觉到：`'+'`、`'-'`、`'*'`、`'/'` 分支之间存在大量重复。这不是坏事，反而说明你已经走到了"下一次代码整理"的最佳时机。
