Lesson 0031

解析函数字面量

scanner 已经认识 fn 了。这一课只做 parser / AST：让匿名函数字面量 fn(x){ x + x } 第一次进入语言表达式系统。

这一步为什么重要： 函数是值——fn(x){ x + x } 是一个表达式，它应该和其他表达式一样能被 let 绑定。这一课先把它的 AST 结构建立起来，函数调用后续再单独推进。

本课范围

  • 在 AST 中新增函数字面量表达式节点 Expr::Fn { params, body }
  • parser 能解析匿名函数字面量 fn(x, y) { x + y }
  • 参数列表可以为空：fn() { 1 + 2 }
  • body 是单个表达式
  • 新增 parser / AST 测试

这一课不做：

  • 不做函数调用 expr(args)
  • 不做 evaluator
  • 不做运行时函数值
  • 不做作用域
  • 不改 scanner

这节课真正新增的知识点

函数字面量是 parser 遇到的另一种 prefix 结构——看到 fn 就知道接下来是参数列表和花括号 body。

和 match 类似：Token::Fn 被 parse_primary() 匹配后，进入一个独立的解析路径，依次消费参数列表、花括号 body。

函数字面量语法

fn ( param , ... ) { body_expr }
  • 关键字 fn
  • 圆括号包裹逗号分隔的参数名列表（可以为空）
  • 花括号包裹单个表达式 body
// 无参数
fn() { 1 + 2 }

// 单参数
fn(x) { x + x }

// 多参数
fn(x, y) { x + y }

推荐的最小实现方向

  1. 在 Expr 中加入 Fn { params: Vec<String>, body: Box<Expr> }
  2. 在 parse_primary() 中新增 Token::Fn 分支
  3. 解析圆括号内的参数名列表（逗号分隔的标识符）
  4. 解析花括号内的 body 表达式
  5. 新增 parser / AST 测试

目标输入 / AST 结构

输入:
fn(x) { x + 1 }

目标 AST:
Expr::Fn {
    params: ["x"],
    body: Expr::Binary {
        left: Expr::Ident("x"),
        op: Token::Plus,
        right: Expr::Number(1),
    }
}
输入:
fn() { 1 + 2 }

目标 AST:
Expr::Fn {
    params: [],
    body: Expr::Binary { ... }
}
