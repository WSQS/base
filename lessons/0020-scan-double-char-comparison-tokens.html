Lesson 0020

Scanner 识别双字符比较运算符

这一课继续停留在 scanner 层，只解决一个新问题：当运算符可能是 1 个字符，也可能是 2 个字符时，scan() 应该如何识别。

这一步为什么重要： 你已经让 scanner 认识了 < 和 >。下一步不是立刻进入 parser，而是继续把比较类 token 在词法层补齐：==、!=、<=、>=。

本课范围

  • 在 Token 中加入双字符比较运算符对应的 token
  • 修改 scan()，支持前瞻并识别 ==、!=、<=、>=
  • 补上 scanner 层面的测试

这一课不做：

  • 不改 parser
  • 不改 evaluator
  • 不让比较表达式真正求值

这节课真正新增的知识点

上一课你只需要根据当前字符做分支。这一课开始，scanner 需要处理：

当前字符不够，需要看下一个字符才能决定 token。

例如：

  • 看到 =，要判断是 = 还是 ==
  • 看到 !，要判断是不是 !=
  • 看到 <，要判断是 < 还是 <=
  • 看到 >，要判断是 > 还是 >=

推荐的最小实现方向

  1. 先决定 token 命名，例如 EqualEqual、BangEqual、LessEqual、GreaterEqual
  2. 把当前逐字符扫描改成能在必要时查看下一个字符
  3. 为这四种双字符 token 各写至少一个 scanner 测试

这节课重点不是“比较语义”，而是“scanner 的前瞻能力”。

目标输入 / 输出

输入:
1 == 2 != 3 <= 4 >= 5

scanner 目标:
[Number(1), EqualEqual, Number(2), BangEqual, Number(3), LessEqual, Number(4), GreaterEqual, Number(5)]

仍然只看 token 序列，不看运行结果。
