Lesson 0011

实现最小 evaluator，让 print 真的输出结果

这一课只教一件事：把已经能 parse 的最小程序真正执行起来，让 print expr; 产生可见输出。

和你的 mission 的关系： 你当前的 Phase 0 路线是 lexer -> parser -> AST -> evaluator。前面三步已经有了雏形，这一课开始补上最后一环，让语言从“能分析”变成“能运行”。

今天只做这一件事

这节课不要一下子把变量环境、错误系统、复杂表达式全做了。范围严格控制在：

  • 能求值 Expr::Number
  • 能求值 Expr::Binary 的 + - * /
  • 能执行 print expr;
  • 暂时跳过 let 的变量保存

也就是说，这一课的最小目标是跑通：

print 1 + 2;

并得到：

3

你现在要建立的新分工

  • scan() 负责把源码拆成 token
  • parse() 负责把 token 组装成 AST
  • eval_expr() 负责把表达式算出一个值
  • eval_program() 负责按顺序执行语句

这一步很重要，因为你开始把“语法结构”和“运行语义”分开了。

建议你这节课先实现的函数

fn eval_expr(expr: &Expr) -> i64
fn eval_program(program: &Program)

先别急着设计太多抽象。最小可运行优先。

推荐的实现顺序

  1. 先让 eval_expr(&Expr::Number(123)) 返回 123
  2. 再让 eval_expr() 支持 Expr::Binary
  3. 在 Binary 里根据 Token::Plus / Minus / Star / Slash 决定怎么计算
  4. 写一个 eval_program()，遍历 program.stmts
  5. 先只处理 Stmt::Print，把表达式结果打印出来

本课目标输入 / 输出

输入程序:
print 1 + 2;

输出:
3
输入程序:
print 8 - 3;

输出:
5
输入程序:
print 6 / 2;

输出:
3

完成标准

  • 你已经实现了一个最小 eval_expr()
  • 你已经能执行 print expr;
  • 你已经用真实输入 / 输出验证至少一个例子

这节课故意不做什么

  • 先不做变量环境，所以 Ident 可以暂时不支持，或者先保留占位行为
  • 先不做 let 的赋值执行
  • 先不做完整错���处理
  • 先不做更复杂表达式优先级修正

这样你才能把注意力集中在“AST 是怎么被执行”的第一条主线。

卡住时优先检查这 4 件事

  • 你是不是把“解析”逻辑和“求值”逻辑混在一起了？
  • 你的 eval_expr() 是不是还没有递归处理左右子表达式？
  • 你是不是一开始就试图支持 let x = ... 的环境存储？
  • 你是不是还没有先拿 print 1 + 2; 做最小验证？
本课后的观察点： 一旦这节课完成，你的语言就第一次具备了“输入源码 → 解析 AST → 执行输出”的真实闭环。哪怕功能还很少，这已经是一门能跑起来的语言原型了。
