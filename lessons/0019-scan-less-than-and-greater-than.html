Lesson 0019

Scanner 先识别 < 和 >

这一课只做词法层的一小步：让 scan() 能把 < 和 > 识别成新的 token。

这一步为什么重要： 你已经明确后续扩展要尽量按单一实现层推进。比较运算符真正进入语言之前，第一步不是 parser，也不是 evaluator，而是先让 scanner 认识它们。

本课范围

  • 在 Token 里加入表示 < 和 > 的新变体
  • 修改 scan()，让它能识别这两个单字符运算符
  • 补上 scanner 层面的测试

这一课不做：

  • 不让 parser 理解比较表达式
  • 不让 evaluator 计算布尔结果
  • 不处理 == 或 !=

你这次真正要学的点

这节课的重点不是“比较表达式”这个大功能，而是：

一个新语言符号，如何先进入 scanner？

也就是说，输入里先出现字符：

<   >

然后 scanner 能先把它们切成 token，后面的 parser 和 evaluator 以后再说。

推荐的最小实现

  1. 在 Token 中加入两个新变体，例如 Less 与 Greater
  2. 在 scan() 的字符分支中加入对 < 与 > 的处理
  3. 仿照现有的 token 测试，新增针对比较符号的 scanner 测试

目标输入 / 输出

输入:
1 < 2 > 3

scanner 目标:
[Number(1), Less, Number(2), Greater, Number(3)]

注意：这里只验证 token 序列，不要求整段程序已经能 parse 成比较表达式。

提醒： 如果你实现过程中发现自己开始想“顺手把 parser 也改了”，先停一下。这节课的胜利条件不是比较表达式跑通，而是 scanner 这一层先独立走通。
