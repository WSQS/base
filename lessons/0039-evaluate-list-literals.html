Lesson 0039

求值 list 字面量

parser 已经能把 [1, 2, 3] 变成 Expr::List。这一课只做 evaluator：让 list 成为运行时值，被 print 输出。

为什么重要： list 走到这一课才算真正"活"。之后可以写 let x = [1, 2, 3]; print(x); 看到输出。这也是语言第五种运行时值类型。

本课范围

  • 给 Value 新增 List(Vec<Value>)
  • 在 eval_expr 中为 Expr::List 添加求值分支
  • 在 print builtin 中支持输出 list 值
  • 新增 evaluator 测试

这一课不做：

  • 不做 list 操作 builtin（len、cons 等）
  • 不改 scanner / parser / AST

这节课的核心知识

list 字面量的求值是递归的——每个元素都是一个表达式，需要逐个求值：

eval_expr(Expr::List(elements), env) => {
    let values = elements.iter()
        .map(|e| eval_expr(e, env))
        .collect();
    Value::List(values)
}

这和字符串求值不一样。字符串是一个原子值——Expr::String("hello") 直接变成 Value::String("hello")。但 list 的每个元素都可能包含子表达式——[1+1, "hello"] 必须先算出 [2, "hello"] 再包装。

推荐的最小实现方向

  1. 在 Value 中加入 List(Vec<Value>)
  2. 在 eval_expr 中为 Expr::List 添加分支
  3. 在 print_builtin 中为 Value::List 添加输出分支
  4. 新增测试

目标输入 / 输出

输入:
let x = [1, 2, 3];
print(x);

期望输出:
[1, 2, 3]
输入:
print([]);

期望输出:
[]

完成标准

  • Value::List(Vec<Value>) 已加入
  • eval_expr(Expr::List(...)) 递归求值并返回运行时 list
  • print 已支持输出 list
  • 新增 evaluator 测试并通过 cargo test
