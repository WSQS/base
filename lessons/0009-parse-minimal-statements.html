Lesson 0009

让 parser 先会解析最小语句骨架

这一课只教一件事：让 parser 真正开始消费 token，并先识别 Phase 0 的两类语句：let 与 print。

和你的 mission 的关系： 你已经完成了 Phase 0 的 lexer，也定义了最小 AST。现在要迈出 parser 的第一步：不再手工构造 AST，而是开始根据 token 类型真正组装出语句结构。

今天只做这一件事

本课不要求你一次性实现完整表达式优先级解析。你只需要先让 parser 会识别两类语句骨架：

  • let name = expr;
  • print expr;

也就是说，这一课的重点是“语句入口判断”，不是“表达式细节全做完”。

本课目标输入 / 输出

输入 token:
Let Ident("x") Equal Number(1) Semicolon

目标 AST:
Stmt::Let { name: "x", expr: Expr::Number(1) }

输入 token:
Print Ident("x") Semicolon

目标 AST:
Stmt::Print { expr: Expr::Ident("x") }

你现在可以暂时把表达式部分保持在最小状态，例如先只接受单个数字或单个标识符，再逐步扩展。

你要亲手完成的步骤

  1. 让 parse() 真正读取 token 序列，而不是直接手工构造固定 AST
  2. 在 parser 入口处先判断当前 token 是 Let 还是 Print
  3. 如果是 Let，至少先读出：关键字、名字、等号、一个最小表达式、分号
  4. 如果是 Print，至少先读出：关键字、一个最小表达式、分号
  5. 先让测试通过最简单的语句输入，再继续扩展表达式解析能力

完成标准

如果你已经做到下面两件事，这节课就完成了：

  • 让 parse() 不再完全手工返回固定 AST，而是开始根据 token 类型分派语句解析
  • 至少能正确解析最简单的 let x = 1; 和 print x; 这类输入

卡住时优先检查这 3 件事

  • 你是不是还没有真正“消费 token”，只是继续手工构造 AST？
  • 你是不是试图一步到位把表达式优先级全部做完？
  • 你是不是把语句入口判断（Let / Print）和表达式解析混成了一坨？
本课后的观察点： 完成这一课后，你的 parser 将第一次真正“工作起来”。虽然它还很小，但它已经不再是占位函数，而是会根据 token 结构构造 AST。
