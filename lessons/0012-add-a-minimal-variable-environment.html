Lesson 0012

加入最小变量环境，让 let 和 ident 真正工作

这一课只教一件事：让解释器记住变量的值，并在之后读出来。目标是让 let x = 1 + 2; print x; 真正输出 3。

和你的 mission 的关系： 你已经打通了最小执行闭环，但目前语言还不能保存状态。Phase 0 想要成为“最小但完整可运行的语言”，变量环境是下一块必不可少的能力：程序不仅要能算，还要能记住结果并再次使用它。

今天只做这一件事

本课只关注“变量能存进去，再读出来”。范围严格控制在：

  • 为 evaluator 增加一个最小环境，例如 HashMap<String, i64>
  • 执行 Stmt::Let 时，把变量名和求值结果存入环境
  • 求值 Expr::Ident 时，从环境中取值
  • 让这条程序跑通：let x = 1 + 2; print x;

这节课故意不做作用域、可变更新、错误恢复。先只做“一个最小表，能存能取”。

你要建立的新关系

  • eval_expr() 不再只看表达式本身，还要读取环境
  • eval_program() 不再只是遍历语句，还要持有并更新环境
  • let 是“写入环境”
  • ident 是“从环境读取”

这一步会让你第一次感受到“运行时状态”是什么。Crafting Interpreters 关于 statements and state 把这件事讲得很清楚：变量的本质就是名字和状态的绑定。

保持最小，不要为了未来先设计复杂运行时结构。当前只需要一个“名字 → 数���”的表就够了。这也符合 Essentials of Compilation 的增量路线：先让整数和变量工作，再逐步扩展更复杂能力。

推荐的实现顺序

  1. 先给 eval_expr() 增加环境参数
  2. 让 Expr::Ident(name) 从环境里找对应值
  3. 让 Stmt::Let 先求值右侧表达式，再把结果写进环境
  4. 让 Stmt::Print 在同一个环境里读取并输出结果
  5. 用 let x = 1 + 2; print x; 做真实验证

本课目标输入 / 输出

输入程序:
let x = 1 + 2;
print x;

输出:
3
输入程序:
let a = 8;
print a;

输出:
8

完成标准

  • 你已经让 Stmt::Let 能把值写进环境
  • 你已经让 Expr::Ident 能从环境读值
  • 你已经用真实输入验证 let x = 1 + 2; print x; 输出 3

这节课故意不做什么

  • 先不做嵌套作用域
  • 先不做重复定义检查
  • 先不做“未定义变量”优雅报错系统
  • 先不做变量重新赋值语法

你的目标不是一次做出完整语言状态模型，而是先掌握“最小环境”这个核心概念。

卡住时优先检查这 4 件事

  • 你是不是给 eval_expr() 传了环境，但递归处理 left / right 时忘了继续传？
  • 你是不是在 let 时把表达式本身存进去了，而不是表达式的值？
  • 你是不是把环境放在了局部临时位置，导致 print 时已经读不到？
  • 你是不是还没先用一个最小例子验证整个链路？
本课后的观察点： 完成这一课后，你的语言将不再只是“马上算完就结束”，而是第一次具备持续状态。程序会开始像真正的小语言那样，能把前面语句的结果留给后面语句使用。
