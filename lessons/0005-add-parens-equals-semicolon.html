Lesson 0005

加入括号 `(` `)`、等号 `=` 和分号 `;`

这一课只教一件事：在你现有的运算符扫描器上，再加入四个常见的单字符 token，让输入更接近真正的小语言语句。

和你的 mission 的关系： 你正在一步步搭一门真实的小语言。今天新增的不是“更复杂的逻辑”，而是让扫描器能识别更像程序语句的基础符号，为后续进入变量定义和语句结构做准备。

今天只新增这 4 个 token

  • LParen 对应 (
  • RParen 对应 )
  • Equal 对应 =
  • Semicolon 对应 ;

暂时仍然不做：标识符、关键字、完整语法分析。

本课目标输入 / 输出

输入: (12 + 3) = 15;
输出: LParen
      Number(12)
      Plus
      Number(3)
      RParen
      Equal
      Number(15)
      Semicolon

你现在仍然只是在做 token 切分，不需要赋予这些符号更多语义。

你要亲手完成的步骤

  1. 在 enum Token 里新增 LParen、RParen、Equal、Semicolon
  2. 在扫描循环里，分别给 '('、')'、'='、';' 添加匹配分支
  3. 沿用你之前的模式：遇到这些单字符 token 时，先结束当前 Number，再推入对应 token
  4. 补充单元测试，覆盖有空格和无空格的输入

完成标准

如果你的代码通过下面两个用例，这节课就完成了：

用例 1:
输入: (12 + 3) = 15;
输出: LParen, Number(12), Plus, Number(3), RParen, Equal, Number(15), Semicolon

用例 2:
输入: (7+8);
输出: LParen, Number(7), Plus, Number(8), RParen, Semicolon

卡住时优先检查这 3 件事

  • 新增符号分支时，是否都沿用了“先结束当前 Number，再推入新 token”的模式？
  • 无空格输入是否也能正确切分？
  • 是否少写了某个新 token 的 enum 变体？
本课后的观察点： 完成这一课后，你的扫描器已经不再只是“表达式符号试验品”，而是开始能切出更像真实语句的 token 流。下一步进入标识符和关键字时，你会明显感觉语言开始真正“长出来”。
