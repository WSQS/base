Lesson 0021

在 parser 中加入比较表达式层

这一课把比较运算符第一次真正接进 parser，但仍然只停留在语法层：目标是构造出正确的 AST，而不是马上求出布尔结果。

这一步为什么重要： scanner 现在已经认识了比较类 token。下一步最自然的推进，不是 evaluator，而是先让 parser 知道这些 token 应该如何进入表达式树，以及它们和加减乘除之间的优先级关系。

本课范围

  • 让 parser 支持比较运算符：<、>、<=、>=、==、!=
  • 给表达式解析增加一层“比较层”
  • 确保比较层优先级低于 + - * /
  • 新增 AST 结构测试

这一课不做：

  • 不让 evaluator 返回布尔值
  • 不让 print 1 < 2; 输出 true
  • 不进入 if 或条件语句

这节课真正新增的知识点

你现在要做的不是“支持比较功能”，而是：

给表达式解析再加一层优先级。

也就是说，parser 需要知道：

  • 乘除比加减更紧
  • 加减比比较运算更紧

所以像：

1 + 2 < 4

应该被解析成：

(1 + 2) < 4

而不是：

1 + (2 < 4)

推荐的最小实现方向

  1. 保留现有 parse_mul_div() 与 parse_add_sub()
  2. 新增一个比较层，例如 parse_comparison()
  3. 让 parse_expr() 从直接调用 parse_add_sub()，改为调用新的比较层
  4. 新增测试，验证 AST 的根运算符已经可以是比较运算符

目标输入 / AST 结构

输入:
print 1 + 2 < 4;

目标：
parser 产出一棵以 Less 为根的 Binary
其 left 是 (1 + 2)
其 right 是 4
输入:
print 2 * 3 == 6;

目标：
parser 产出一棵以 EqualEqual 为根的 Binary
其 left 是 (2 * 3)
其 right 是 6
