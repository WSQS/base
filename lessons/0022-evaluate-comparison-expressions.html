Lesson 0022

在 evaluator 中计算比较表达式

这一课把比较表达式从“只是 AST”继续推进到“真正求值”：当左右两边是整数时，比较运算应该返回 Value::Boolean。

这一步为什么重要： 第 18 课引入 Value 的意义，到这一课会第一次真正兑现。现在 evaluator 不再总是返回整数，比较表达式将开始返回布尔结果。

本课范围

  • 在 eval_expr() 中支持比较运算符：<、>、<=、>=、==、!=
  • 当左右两边都是整数时，返回 Value::Boolean(true/false)
  • 补上 evaluator 测试
  • 补上最小程序级验证，例如 print 1 < 2; 输出 true

这一课不做：

  • 不支持布尔字面量 true / false
  • 不支持逻辑运算 && / ||
  • 不进入 if 或条件语句

这节课真正新增的知识点

你现在要面对的关键变化是：

同样是 Expr::Binary，不再总返回 Integer，也可能返回 Boolean。

这意味着 evaluator 的二元运算分支需要开始按运算符类型区分：

  • 算术运算：返回 Value::Integer
  • 比较运算：返回 Value::Boolean

推荐的最小实现方向

  1. 保留当前整数算术分支
  2. 在整数-整数的二元运算匹配中，加入比较运算符处理
  3. 让比较结果包装成 Value::Boolean(...)
  4. 新增测试，验证比较求值结果和打印输出

这节课重点不是引入更多语法，而是第一次真正让 Value::Boolean 参与运行时。

目标输入 / 输出

输入:
print 1 + 2 < 4;
print 2 * 3 == 6;
print 5 != 5;

目标输出:
true
true
false

如果 evaluator 做对了，当前语言就第一次拥有了真正的布尔运行时结果。
