Lesson 0028

解析字符串字面量

上一课 scanner 已经能产出 Token::String(...)。这一课只做 parser / AST：把字符串 token 接进表达式系统，变成字符串字面量节点。

这一步为什么重要： scanner 只负责把原始字符切成 token。真正让字符串进入语言表达式系统，还需要 parser 知道：Token::String("hello") 应该变成什么 AST 节点。

本课范围

  • 在 AST 中新增字符串字面量表达式节点
  • 让 parse_primary() 能识别 Token::String(...)
  • 新增 parser / AST 测试

这一课不做：

  • 不改 evaluator
  • 不改 Value
  • 不支持字符串运算
  • 不支持字符串 comparison
  • 不支持字符串 pattern

这节课真正新增的知识点

一个新字面量进入语言，通常要经过三层：

  1. scanner 产出 token
  2. parser 把 token 变成 AST 节点
  3. evaluator 把 AST 节点变成运行时值

你已经完成了第 1 步；这节课只完成第 2 步。

Token::String("hello") -> Expr::String("hello")

推荐的最小实现方向

  1. 在 Expr 中加入字符串字面量变体，例如 Expr::String(String)
  2. 在 parse_primary() 中新增 Token::String(s) 分支
  3. 保持字符串只作为最小 primary expression，不扩展其他规则
  4. 新增测试，断言 parse 后得到字符串 AST 节点

目标输入 / AST 结构

输入:
print "hello";

目标：
得到一条 Print 语句
其中 expr = Expr::String("hello")
输入:
let name = "world";

目标：
得到一条 Let 语句
其中 expr = Expr::String("world")

完成标准

  • Expr 中已新增字符串字面量节点
  • parse_primary() 已支持 Token::String(...)
  • 新增 parser / AST 测试并通过
  • evaluator / runtime value 暂时不改
提醒： 这节课的胜利条件不是字符串已经能运行，而是“字符串已经进入 AST”。
参考来源：
