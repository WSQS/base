Lesson 0032

解析函数调用

第 31 课函数字面量已经进入 AST。这一课只做函数调用 expr(args) 的解析——让语言第一次拥有调用表达式的语法结构。

这一步为什么重要： 函数必须是值，调用必须是表达式。函数字面量在第 31 课已经是表达式了，现在还需要让 add(1, 2)、(fn(x){ x + x })(3) 也成为表达式，才能写出真正可调用的程序。

本课范围

  • 在 AST 中新增函数调用表达式节点 Expr::Call { func, args }
  • parser 能解析 expr(args) 这种 postfix 调用结构
  • 实参列表可以为空：f()，也可以有多实参：f(1, 2)
  • func 是任意表达式，不限于标识符
  • 新增 parser / AST 测试

这一课不做：

  • 不做 evaluator
  • 不做运行时函数值
  • 不做作用域
  • 不改 scanner

这节课真正新增的知识点

函数调用是一种 postfix 结构——它发生在解析完一个基础表达式之后，继续往前看发现有一个 ( 时才被触发。

这和你的 parser 里已经有的一种模式完全一样：parse_mul_div 解析完 primary 后检查 * 或 /，parse_add_sub 检查 + 或 -。函数调用也是 "解析完一个表达式 + 看到某个 token 后继续消费更多内容"。唯一的区别是：调用的 postfix 触发 token 是 LParen。

调用表达式的 postfix 位置

调用应该放在 primary 层还是更高层？

  • 放在 primary 层（在 parse_primary 解析完基础表达式后检查）——最自然，因为 f(1) 中的 f 本身是一个 primary，这就是一个 post-primary 检查
  • 放在更高层（如单独一个 parse_call 层在 primary 之上）——更灵活，允许 1 + f(1) 中的 f(1) 作为整个 term 参与运算

建议实现：在 parse_primary 解析完基础表达式后加一个 postfix 循环，检查后续 token 是否为 LParen，如果是就进入调用参数解析。这样 f(1)(2)（链式调用）也能自然支持。

推荐的最小实现方向

  1. 在 Expr 中加入 Call { func: Box<Expr>, args: Vec<Expr> }
  2. 在 parse_primary() 末尾（返回基础表达式之前），加一个 while 循环检查当前 token 是否为 LParen
  3. 如果是，消费 (，解析逗号分隔的实参列表，消费 )，用 Expr::Call 包裹当前表达式
  4. 循环继续检查下一个 token 是否还是 LParen（支持链式调用）
  5. 新增 AST 测试

目标输入 / AST 结构

输入:
add(1, 2)

目标 AST:
Expr::Call {
    func: Expr::Ident("add"),
    args: [
        Expr::Number(1),
        Expr::Number(2),
    ]
}
输入:
f()

目标 AST:
Expr::Call {
    func: Expr::Ident("f"),
    args: [],
}
输入:
(fn(x) { x + x })(3)

目标 AST:
Expr::Call {
    func: Expr::Fn { params: ["x"], body: Binary { ... } },
    args: [
        Expr::Number(3),
    ]
}
