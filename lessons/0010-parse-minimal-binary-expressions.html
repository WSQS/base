Lesson 0010

让 parser 先会解析最小二元表达式

这一课只教一件事：把 parser 的表达式能力从单个 Number / Ident，扩展到最小的二元表达式结构。

和你的 mission 的关系： 你已经让 parser 能识别最小语句骨架，但 Phase 0 的语言不是只包含单个原子表达式。为了真正接近可运行闭环，parser 现在必须开始把 1 + 2 这类输入组装成结构化表达式树。

今天只做这一件事

不要一次性做完整优先级系统。本课的目标很小：

  • 至少能把 1 + 2 解析成 Expr::Binary
  • 至少能把 x + 3 解析成 Expr::Binary
  • 把这个能力接进已有的 let / print 语句解析中

也就是说，你先做“最小 binary expression parser”，而不是完整表达式系统。

本课目标输入 / 输出

输入: let x = 1 + 2;
目标 AST:
Stmt::Let {
  name: "x",
  expr: Expr::Binary {
    left: Box::new(Expr::Number(1)),
    op: Token::Plus,
    right: Box::new(Expr::Number(2)),
  }
}

输入: print x + 3;
目标 AST:
Stmt::Print {
  expr: Expr::Binary {
    left: Box::new(Expr::Ident("x")),
    op: Token::Plus,
    right: Box::new(Expr::Number(3)),
  }
}

你要亲手完成的步骤

  1. 先保留你已经会的“读取单个 Number / Ident”能力
  2. 在此基础上，检查后面是否紧跟一个二元运算符（先从 + 开始就够了）
  3. 如果有，就继续读右侧的一个最小原子表达式
  4. 把左、操作符、右组装成 Expr::Binary
  5. 让这个最小表达式解析能力接进已有的 let / print 语句解析

完成标准

如果你已经做到下面两件事，这节课就完成了：

  • parser 能把最小二元表达式组装成 Expr::Binary
  • 这个能力已经接进 let 或 print 语句解析中

卡住时优先检查这 3 件事

  • 你是不是还只会读取单个 Number / Ident，没有往后看操作符？
  • 你是不是把 token 顺序消费和 AST 组装写混了？
  • 你是不是已经开始试图一次支持所有优先级和括号规则？
本课后的观察点： 完成这一课后，parser 就不再只是“语句骨架解析器”，而会第一次真正构造出表达式树。这会让后续 evaluator 的目标也开始清晰起来。
