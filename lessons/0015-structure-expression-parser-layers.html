Lesson 0015

把表达式 parser 拆成清晰层次

这一课只教一件事：不是继续加新语法，而是把你已经会解析的表达式能力，整理成更清晰的分层结构。

和你的 mission 的关系： 你已经完成最小闭环，也已经支持括号和最小优先级。现在最重要的不再是继续堆功能，而是把现有表达式 parser 整理得更清晰、更稳，符合你新的 mission：小步扩展，并及时整理结构，为长期自举路径积累更好的核心骨架。

今天只做这一件事

这节课不引入新 token、新语句，也不增加新 evaluator 能力。范围严格控制在：

  • 把表达式解析拆成更清楚的层次
  • 至少区分：原子表达式 / 乘除层 / 加减层
  • 保留现有括号分组能力
  • 保留现有最小优先级能力

这节课的目标不是“多支持一种输入”，而是让你现在已经支持的输入，能由一个更稳的 parser 结构来承载。

你现在最值得建立的结构直觉

表达式 parser 不应该主要靠一个布尔值和很多条件分支勉强撑住。更稳的思路是：

parse_expr()
  -> parse_add_sub()
      -> parse_mul_div()
          -> parse_primary()

也就是：

  • parse_primary()：数字、标识符、括号表达式
  • parse_mul_div()：处理 * /
  • parse_add_sub()：处理 + -

这比把括号、原子、优先级分支都揉进同一个函数里更适合继续成长。这个分层思想和 Crafting Interpreters、precedence climbing 的核心直觉是一致的：表达式要按层处理。

推荐的最小整理目标

  1. 先把“读数字/标识符/括号表达式”集中到一个入口
  2. 让乘除层只关心如何把多个 primary 拼成更紧的 binary
  3. 让加减层只关心如何在乘除层结果上继续拼接
  4. 让总入口只负责调用最外层表达式解析

注意：这一课不是追求“最短代码”，而是追求“职责更清楚”。

本课目标输入 / 输出

输入程序:
print 1 + 2 * 3;

目标：
仍然输出 7
但表达式解析结构更清楚
输入程序:
print (1 + 2) * 3;

目标：
仍然输出 9
括号和优先级同时成立

完成标准

  • 你已经把表达式解析拆成更清楚的层次
  • 你仍然保留了括号和最小优先级行为
  • 你能说清每一层负责的范围

这节课故意不做什么

  • 先不加入新 token 或新语句
  • 先不引入完整 Pratt parser 理论实现
  • 先不同时整理 scanner 和 evaluator
  • 先不追求“最优雅最终版”结构

这一课的目标只是把当前表达式 parser 从“能用但混乱”往“能用且更清晰”推进一步。

卡住时优先检查这 4 件事

  • 你是不是还在让一个函数同时处理括号、原子、优先级和右侧递归？
  • 你是不是已经拆层了，但每层职责仍然不清楚？
  • 你是不是整理结构时把已有测试能力弄坏了？
  • 你是不是一上来就想做完整通用 parser 框架，而不是先做当前最小整理？
本课后的观察点： 完成这一课后，你的 parser 结构会比现在更适合继续成长。这个变化不一定立刻带来“更多语法”，但会显著降低后面扩展表达式能力时的混乱程度。
