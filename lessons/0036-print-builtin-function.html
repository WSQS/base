Lesson 0036

print 作为 builtin 函数

第 35 课已经把 print 从关键字降为普通标识符，表达式语句已就绪。这一课只做 evaluator：注入 print 作为 builtin，让 print("hello") 真正运行。

这一步为什么重要： 这是"print 降级"的最后一步。做完后，print 和其他 builtin（add 等）在运行时地位完全相同——都是全局环境中的 Value::BuiltinFn。语句系统也在此定型：let + 表达式语句。

本课范围

  • 实现 print_builtin：接收一个参数，打印后返回该值
  • 在 eval_program() 初始环境中注入 "print" → Value::BuiltinFn(print_builtin)
  • 将 Stmt::Expr 的分支改为纯粹的求值（不再自动打印结果）
  • 清理掉当前 Stmt::Expr 中残留的旧的打印逻辑
  • 清理 evaluator 中的 unreachable pattern warning

这一课不做：

  • 不改 scanner / parser / AST
  • 不做其他 builtin
  • 不做多参数 print

这节课的核心知识

1. print builtin 的实现

print 接收一个参数，按类型格式化输出，返回原值：

fn print_builtin(args: Vec<Value>) -> Value {
    match &args[0] {
        Value::Integer(i) => print!("{i}\n"),
        Value::Boolean(b) => print!("{b}\n"),
        Value::String(s) => print!("{s}\n"),
        _ => print!("{:?}\n", args[0]),
    }
    args[0].clone()  // 返回原值，让 print 也可以作为表达式的一部分
}

2. Stmt::Expr 改为纯求值

当前 Stmt::Expr 分支内还是旧的打印逻辑。需要简化为：

// 旧（第 35 课临时状态）
Stmt::Expr { expr } => {
    let result = eval_expr(expr, &env);
    match result {
        Value::Integer(i) => print!("{i}\n"),
        // ...
    }
}

// 新（本课）
Stmt::Expr { expr } => {
    eval_expr(expr, &env);  // 求值，丢弃结果
}

3. 环境注入

在 eval_program() 中注入 print：

pub fn eval_program(program: &Program) {
    let mut env = HashMap::new();
    env.insert("print".to_string(), Value::BuiltinFn(print_builtin));
    eval_program_with_env(program, &mut env);
}

推荐的最小实现方向

  1. 在 eval.rs 中实现 print_builtin 函数
  2. 将 Stmt::Expr 分支简化为纯求值（删除 print! 及其 match）
  3. 在 eval_program() 中注入 print 到初始环境
  4. 清理 evaluator 中的 unreachable pattern warning
  5. 运行测试，确认 print(x) 形式的测试仍然通过

目标输入 / 输出

输入:
let x = 42;
print(x);

期望输出:
42
输入:
print("hello");

期望输出:
hello
输入:
let result = print(1 + 2);
print(result);

期望输出:
3      ← 第一次 print 输出
3      ← 第二次 print 返回原值，再输出
