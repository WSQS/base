Lesson 0035

移除 print 语句，引入表达式语句

print 不再是特殊关键字，语言简化为：只有 let 声明和表达式语句两种语句。这是一次语法统一——让语言更均匀。

这一步为什么重要： 去掉了语言中最后一个"纯副作用语句"。从此任何表达式都可以作为语句执行（1 + 2;、f(1);），print 只是一个普通标识符——下一课才作为 builtin 函数注入。这为将来"所有语句都可以有值"铺路。

本课范围

  • scanner：移除 "print" 关键字识别
  • AST：移除 Stmt::Print，新增 Stmt::Expr { expr: Expr }
  • Token::Print 保留但不再使用（删掉也行，维护成本最低）
  • parser：移除 Token::Print 特殊分支，新增表达式语句解析
  • 迁移全部现有测试：print x; → 不再有效；add(1, 2); 成为表达式语句
  • 新增测试

这一课不做：

  • 不做 evaluator 改动
  • 不注入 print builtin
  • 不改 eval_program_with_env（Stmt::Print 分支暂时保留但不可达——下一课处理）

这节课的核心知识：语句最小化

在 print 是关键字时，语言的语句系统是：

let x = expr;
print expr;

移除之后变成：

let x = expr;
expr;

任何表达式后面加分号都是合法语句。这是**表达式优先**语言的方向——和对"函数是值"的设计一致。

Parser 改动

在 parse() 里，当看到 Token::Let 时走现有的 let 路径，其余情况默认作为表达式语句：

match t {
    Token::Let => { /* 现有的 let 解析 */ }
    _ => {
        // 表达式语句：解析一个表达式，期待分号结尾
        let expr = parse_expr(tokens, i);
        // parse_expr 已经消费了分号那边的错误日志
        i += 1;  // 跳过分号
        result.stmts.push(Stmt::Expr { expr });
    }
}

推荐的最小实现方向

  1. 在 Stmt 中加入 Expr { expr: Expr }
  2. 移除 scanner 中 "print" → Token::Print 的两处映射
  3. 移除 parser 的 Token::Print 分支
  4. 在 parser 的 _ => 兜底分支中实现表达式语句解析
  5. 迁移测试：所有 print x; 替换为不依赖 print 语句的形式
  6. 新增测试（例如 let x = 1 + 2; 用表达式语句验证 add(1, 2); 被解析）

目标迁移示例

// 旧形式（本课后不再有效）
print x;

// 旧形式
print 1 + 2;

// 新测试应改用环境验证
let x = add(1, 2);  // 使用 builtin add 验证结果
let y = 1 + 2;      // 直接表达式语句不产出值，用 let 绑定验证

完成标准

  • scanner 已移除 print 关键字
  • Stmt::Expr { expr } 已加入 AST
  • parser 已移除 Token::Print 分支，新增表达式语句解析

提醒： 这节课做完后 print 只是普通标识符。下一课才是 evaluator：注入 print builtin + 处理表达式语句求值。两课之间的短暂空窗是设计上可预期的。
