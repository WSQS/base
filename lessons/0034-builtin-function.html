Lesson 0034

支持 builtin 函数

函数调用已经能跑了，但所有函数都必须用户定义。这一课只做 evaluator：让语言第一次拥有内置函数——由 Rust 原生实现、在环境初始化时注入的函数。

这一步为什么重要： builtin 函数是语言的"原生能力接口"。用户定义的函数只是组合已有的操作，而 builtin 可以做到语言语法层面做不到的事——比如类型检查、字符串操作、I/O。第一个 builtin 成功后，后续添加新 native 能力只需一行代码。

本课范围

  • 给 Value 新增 builtin 函数变体
  • 在 eval_expr(Expr::Call) 中识别并执行 builtin
  • 在环境初始化时预注入第一个 builtin：add(a, b)
  • 新增 evaluator 测试

这一课不做：

  • 不做多个 builtin（先把基础设施做稳）
  • 不做类型错误处理（builtin 内部可以先简单断言）
  • 不改 scanner / parser / AST

这节课的核心知识

1. Builtin 值类型

Value::Fn 存的是参数名 + body 表达式——调用时创建局部环境、在环境中求值。

Value::Builtin 存的是一个 Rust 函数指针——调用时直接把已求值的实参传进去，由 Rust 代码执行。

// Builtin 签名
Value::Builtin(fn(Vec<Value>) -> Value)

// 一个具体的 builtin 例子
fn add_builtin(args: Vec<Value>) -> Value {
    match (&args[0], &args[1]) {
        (Value::Integer(a), Value::Integer(b)) => Value::Integer(a + b),
        _ => { log!("add expects two integers"); Value::Integer(0) }
    }
}

2. 调用时的分派

在 Expr::Call 求值时，求值 func 之后加一个 match：

match fun {
    Value::Fn { params, body } => {
        // 现有的逻辑：克隆环境、绑定参数、求值 body
    }
    Value::Builtin(f) => {
        // 新逻辑：求值所有实参 → 传给 f → 直接返回结果
        let args_vals: Vec<Value> = args.iter()
            .map(|a| eval_expr(a, env))
            .collect();
        f(args_vals)
    }
    _ => { ... }
}

3. 环境初始化注入

在 eval_program() 创建初始 env 后，注入 builtin：

pub fn eval_program(program: &Program) {
    let mut env = HashMap::new();
    env.insert("add".to_string(), Value::Builtin(add_builtin));
    eval_program_with_env(program, &mut env);
}

推荐的最小实现方向

  1. 在 Value 中加入 Builtin(fn(Vec<Value>) -> Value)
  2. 在 Expr::Call 的 match fun 中新增 Value::Builtin(f) 分支
  3. 实现 add_builtin 函数
  4. 在 eval_program 中注入 add 到初始环境
  5. 新增测试

目标输入 / 输出

输入:
let x = add(1, 2);
print x;

期望输出:
3
输入:
let a = add(10, add(3, 4));
print a;

期望输出:
17
