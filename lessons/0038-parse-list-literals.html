Lesson 0038

解析 list 字面量

scanner 已认识 [ ]。这一课只做 parser / AST：让 [1, 2, 3] 变成 list 字面量表达式节点。

为什么重要： list 是自举所需的最基础数据结构。tokens 列表、AST 节点列表、函数参数列表——这些在自举实现里到处都需要。list 字面量是 entry point，parser 先认识它，evaluator 后续才能把它变成运行时值。

本课范围

  • 在 AST 中新增 Expr::List(Vec<Expr>)
  • 在 parse_primary() 中新增 Token::LBracket 分支
  • 解析逗号分隔的元素列表，支持空 list []
  • 新增 parser / AST 测试

这一课不做：

  • 不做 evaluator
  • 不做 Value::List
  • 不做 list 操作 builtin
  • 不改 scanner

这节课真正新增的知识点

list 字面量是一个 prefix 结构——看到 LBracket 后进入独立的解析路径，和 Token::Fn、Token::Match 的模式一样。

解析逻辑和函数实参列表极为相似：逗号分隔的表达式序列，以右定界符结束。不同之处在于 list 允许空列表 []。

推荐的最小实现方向

  1. 在 Expr 中加入 List(Vec<Expr>)
  2. 在 parse_primary() 中新增 Token::LBracket 分支
  3. 若下一个 token 是 RBracket → 空 list
  4. 否则循环解析逗号分隔的表达式，直到遇到 RBracket
  5. 新增测试

目标输入 / AST 结构

输入:
[1, 2, 3]

目标 AST:
Expr::List([
    Expr::Number(1),
    Expr::Number(2),
    Expr::Number(3),
])
输入:
[]

目标 AST:
Expr::List([])
输入:
["hello", 42, true]

目标 AST:
Expr::List([
    Expr::String("hello"),
    Expr::Number(42),
    Expr::Boolean(true),
])

完成标准

  • Expr::List(Vec<Expr>) 已加入 AST
  • parser 已支持 [elem, ...] 和空 list []
  • 元素可以是任意表达式（数字、字符串、布尔、函数字面量等）
  • 新增 parser / AST 测试并通过
  • scanner / evaluator 暂时不改
提醒： 这节课的胜利条件是 [1, 2, 3] 被解析为 Expr::List([...])。
