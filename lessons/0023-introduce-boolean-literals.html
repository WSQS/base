Lesson 0023

引入布尔字面量 true / false

这一课先补齐布尔值在源语言中的直接入口：让语言不仅能“算出布尔值”，还可以直接写出布尔字面量。

这一步为什么重要： 当前 runtime 已经有 Value::Boolean，比较表达式也已经能返回布尔结果。但源语言还不能直接写 true / false。这会让后续的模式匹配和控制流设计显得不对称，所以应该先补上布尔字面量。

本课范围

  • scanner 识别关键字 true 与 false
  • parser 能把它们解析成布尔字面量表达式
  • evaluator 返回 Value::Boolean
  • 补上测试，验证布尔字面量能单独使用，也能进入环境与输出

这一课不做：

  • 不做 match
  • 不做 if
  • 不做逻辑运算 && / ||
  • 不做布尔与整数之间的隐式转换

这节课真正新增的知识点

你现在要补的不是新的控制流，而是：

让 Boolean 像 Integer 一样，拥有自己的源语言字面量入口。

也就是说，当前语言终于可以直接写：

true
false

而不必只能依赖比较表达式间接得到布尔值。

推荐的最小实现方向

  1. 在 token 层把 true / false 当作关键字，而不是普通标识符
  2. 在 AST 中加入布尔字面量表达式节点
  3. 让 eval_expr() 在遇到布尔字面量时直接返回 Value::Boolean
  4. 新增最小测试：布尔字面量求值、打印、进入环境

目标输入 / 输出

输入:
print true;
print false;
let x = true;
print x;

目标输出:
true
false
true

这一课完成后，布尔值将不再只是比较表达式的副产物，而是语言里的一级字面量。
