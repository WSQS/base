Lesson 0002

亲手实现只支持数字的最小扫描器

这一课只教一件事：把“字符串输入”变成 `NUMBER` token 输出。你不会一次实现完整 lexer，只会亲手搭起最小第一步。

和你的 mission 的关系： 你要亲手实现自己的语言。真正的入口不是先讨论全部语法，而是先实现一个极小但真实的扫描器：输入文本，输出 token。今天只做数字，建立第一块可运行的地基。

今天只做这一件事

  • 输入：一段只包含整数字面量和空白的文本
  • 输出：按顺序得到一串 NUMBER token
  • 暂时不做：+、标识符、关键字、括号、报错体系
目标输入 / 输出示例： 
输入:  12  7 900
输出: NUMBER(12)
      NUMBER(7)
      NUMBER(900)

你要亲手完成的最小程序

请你自己创建一个最小 Rust 程序，并让它完成下面的行为：

1. 读取一个字符串
2. 从左到右扫描字符
3. 跳过空白
4. 如果遇到数字，就继续向后读，直到不是数字为止
5. 把这一段数字输出成 NUMBER(...)

你现在不需要设计通用架构，只需要把这个最小逻辑跑通。

实践步骤

Step 1

先新建一个 Rust 小程序，并把输入先写死成：

"123"

Step 2

让程序输出：

NUMBER(123)

Step 3

把输入改成：

"7 42 900"

再扩展逻辑，要求输出三行 token。

Step 4

确认程序能跳过多个空格，而不是只处理一个空格。

互动检查 1：你应该输出什么？

如果输入是 "88 5 701"，正确输出应该是哪一项？



互动检查 2：你漏掉了哪一步？

如果你的程序只能处理 "7 42"，但遇到 "7 42" 就出错，最可能缺了什么？



自检：贴出你的程序输出

当你用输入 "12 34 5" 运行你自己的程序后，把你的实际输出粘贴到下面，和目标对比。

完成标准

如果你已经自己写出一个 Rust 程序，并且它对下面输入给出正确输出，这节课就完成了：

输入:  "12 34 5"
输出: NUMBER(12)
      NUMBER(34)
      NUMBER(5)

卡住时优先检查这 3 件事

  • 是不是只读了一个字符，而没有持续读取整个数字？
  • 是不是没有循环跳过多个空白？
  • 是不是读完一个数字后，没有继续扫描后面的字符？
