Lesson 0026

让 evaluator 执行最小 match 表达式

parser 已经能把 match 变成 AST 树了。这一课只做一件事：让 evaluator 真正跑起来 Expr::Match，让 match 第一次产出计算结果。

这一步为什么重要： 这是你的语言第一次拥有真正意义上的控制流——根据一个值的不同，选择不同的分支结果。在这之前，所有表达式都是"算出来就是那个值"，而 match 第一次让程序"做选择"。

本课范围

  • 在 eval_expr 中为 Expr::Match 添加求值分支
  • 先求值 subject 表达式，拿到一个 Value
  • 按顺序逐个尝试每个 arm 的 pattern 是否匹配
  • 第一个匹配成功的 arm，对其 body 表达式求值并返回结果
  • _ 通配模式匹配任何值
  • 新增 evaluator 测试

这一课不做：

  • 不做变量绑定模式（如 x => ... 绑定名字）
  • 不做 guard（如 x if x > 0 => ...）
  • 不做穷尽性检查（编译期判断是否所有情况都覆盖）
  • 不改 scanner，不改 parser，不改 AST 结构

核心知识：模式匹配的求值规则

match 的求值其实只有两条简单规则：

  1. 先算 subject，再匹配：先对 match 后面那个表达式求值，得到一个具体的值。然后拿这个值去逐个 arm 试。
  2. 从上到下，第一个匹配就返回：按 arm 书写顺序逐个检查 pattern。第一个能匹配的 arm 胜出，对它的 body 求值，整个 match 的结果就是它。_ 匹配任何值，通常放在最后当兜底。
// 伪代码描述求值逻辑
let subject_value = eval(subject, env);
for arm in arms:
    if pattern_matches(arm.pattern, subject_value):
        return eval(arm.body, env);
// 如果没有 arm 匹配，返回一个默认值或报错

模式如何匹配值

这一课只有三种 pattern，匹配规则非常直接：

  • Pattern::Number(n) 匹配 Value::Integer(v) 当且仅当 n == v
  • Pattern::Boolean(b) 匹配 Value::Boolean(v) 当且仅当 b == v
  • Pattern::Wildcard 匹配任何 Value，永远成功

类型不同（比如拿 Number(1) 去匹配布尔值）就是不匹配，跳过这个 arm 继续看下一个。

推荐的最小实现方向

  1. 在 eval_expr 的 match 中，把当前 _ => { ... } 兜底分支替换为真正的 Expr::Match 处理
  2. 先求值 subject，得到 Value
  3. 遍历 arms，对每个 arm 调用一个模式匹配辅助函数
  4. 第一个匹配成功的 arm，对其 body 求值并 return
  5. 如果全部不匹配，可以返回一个默认值或 log! 报错
  6. 写测试验证

目标输入 / 输出

输入:
print match 2 { 1 => 100, 2 => 200, _ => 0 };

期望输出:
200
输入:
let x = 1 + 1;
print match x { 1 => 100, 2 => 200, _ => 0 };

期望输出:
200
输入:
print match true { true => 1, false => 0 };

期望输出:
1
