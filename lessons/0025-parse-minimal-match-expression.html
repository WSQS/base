Lesson 0025

让 parser 识别最小 match 表达式外壳

这一课不做完整匹配求值，只把 match 作为表达式外壳第一次接进 parser，并且引入独立的 Pattern AST。

这一步为什么重要： scanner 现在已经认识了 match 相关 token。下一步最自然的推进，不是立刻进入 evaluator，而是先让 parser 知道：match expr { pattern => expr, ... } 这种结构到底如何进入 AST。

本课范围

  • 在 AST 中新增独立的 Pattern 节点
  • 在 AST 中新增最小 MatchArm 和 Expr::Match（或等价结构）
  • parser 能识别最小 match 表达式外壳
  • 第一版只支持这些模式进入 AST：true、false、整数字面量、_
  • 新增 AST / parser 测试

这一课不做：

  • 不做 match 的求值执行
  • 不做变量绑定模式
  • 不做 guard
  • 不做穷尽性检查
  • 不做多语句 arm

这节课真正新增的知识点

这节课最关键的不是语法字符，而是 AST 分层：

Pattern 不属于 Expr，但 Match 表达式会同时包含 subject 表达式和 arm 模式列表。

也就是说，parser 第一次要同时处理：

  • 一个待匹配的表达式 subject
  • 一组模式分支 arm
  • 每个 arm 的 body expression

推荐的最小实现方向

  1. 先定义最小 Pattern 枚举，只放第一版真的要支持的几种模式
  2. 定义最小 MatchArm 结构
  3. 在 Expr 中加入 Match 变体
  4. 给 parser 增加最小 match 外壳解析逻辑
  5. 新增测试，验证一段 match 输入已能被构造成正确 AST

目标输入 / AST 结构

输入:
print match true {
    true => 1,
    false => 0,
    _ => 2,
};

目标：
parser 产出一棵 Expr::Match（或等价节点）
其中：
- subject = Expr::Boolean(true)
- arms = [true => 1, false => 0, _ => 2]

这一课只要求“树搭对”，不要求已经算出结果。

提醒： 这节课的胜利条件不是模式匹配已经能运行，而是“match 的语法外壳和 AST 结构已经进入语言”。
参考来源：
  • OCaml Manual — Patterns：模式是独立语法类别，而不是表达式的一个子集。
  • Haskell 2010 Report — Case Expressions and Pattern Matching：`case` 这类匹配内核先作为表达式进入语言，是非常经典的路线。

如果这一步做稳，下一课再进入 evaluator 的 match 求值时，层次就会更清楚。
