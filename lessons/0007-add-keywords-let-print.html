Lesson 0007

把 let 和 print 从名字中区分出来

这一课只教一件事：当扫描器读到一个名字后，判断它是普通标识符，还是 Phase 0 里的保留关键字。

和你的 mission 的关系： 你已经让扫描器第一次学会识别名字。现在要完成 Phase 0 的 lexer 收尾：让它不仅知道“这是名字”，还知道“这个名字是不是语言自己保留的词”。这样后面的 parser 才能自然识别 let 语句和 print 语句。

今天只新增这 2 个 token

  • Let
  • Print

目标不是发明新扫描策略，而是在已经能读出 Ident(String) 的基础上，再判断扫描出的名字是不是关键字。

本课目标输入 / 输出

输入: let x = 12;
输出: Let
      Ident("x")
      Equal
      Number(12)
      Semicolon

输入: print foo;
输出: Print
      Ident("foo")
      Semicolon

也就是说，let 和 print 不再是普通 Ident，而要变成专门 token。

你要亲手完成的步骤

  1. 在 enum Token 中新增 Let 和 Print
  2. 保持现有标识符扫描逻辑不变，先把连续名字读完整
  3. 在输出名字 token 的那一步，判断字符串内容是否等于 "let" 或 "print"
  4. 如果匹配关键字，就输出对应 token；否则仍输出 Ident(...)
  5. 补充测试，验证关键字和普通标识符都能正确区分

用例 2:

卡住时优先检查这 3 件事

  • 是不是先把整个名字读完，再判断是否是关键字？
  • 是不是把 let 和 print 错误地仍然输出成了 Ident？
  • 普通名字如 foo、bar 是否仍然保留为 Ident？
本课后的观察点： 完成这一课后，你的 lexer 就基本收尾了。接下来 parser 不必再去猜测 Ident("let") 到底是不是关键字，而可以直接根据 token 类型解析语句。
