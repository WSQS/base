Lesson 0037

扫描 list token

进入 Phase 2，第一件事是引入 list 数据结构。这一课只做 scanner：让语言第一次认识方括号 [ ]。

为什么从这里开始： list 字面量语法是 [1, 2, 3]。scanner 已经认识逗号，现在只需要让方括号也变成独立 token。这一步完成后，后续 parser 才能把 [1, 2, 3] 构造成 list 字面量 AST。

本课范围

  • 给 Token 新增 LBracket 和 RBracket
  • 在 scanner 中为 [ 和 ] 添加扫描分支
  • 新增 scanner 测试

注意：{ } 现在是 LBrace / RBrace（用于函数体和 match），[ ] 作为新 token 对——命名用 Bracket 区分。

这一课不做：

  • 不改 parser
  • 不改 AST
  • 不改 evaluator
  • 不做 list 字面量解析

这节课真正新增的知识点

scanner 对单字符 token 的处理模式已经很成熟了：看到字符 → flush → push token。方括号和之前的 +、-、(、)、{、} 完全一样。

唯一要注意的是命名——{ } 已经有 LBrace / RBrace，所以 [ ] 用 LBracket / RBracket。

目标输入 / token 输出

输入:
[1, 2, 3]

目标 token:
[
  LBracket,
  Number(1),
  Comma,
  Number(2),
  Comma,
  Number(3),
  RBracket,
]
输入:
[]

目标 token:
[
  LBracket,
  RBracket,
]

推荐的最小实现方向

  1. 在 Token 中加入 LBracket 和 RBracket
  2. 在 scanner 的字符匹配分支中添加 '[' 和 ']'
  3. 新增 scanner 测试

互动检查：为什么不是 LSquare / RSquare？



完成标准

  • Token 中已新增 LBracket 和 RBracket
  • scanner 已支持 [ 和 ]
  • 新增 scanner 测试并通过
  • parser / AST / evaluator 暂时不改
提醒： 这节课的胜利条件不是 list 已经能运行，而是"scanner 已经认识方括号"。
有疑问？ 随时向 agent 提问。scanner 加单字符 token 已有标准模式，和在 [ 里继续跟 { 类比就行。
