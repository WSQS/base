Reference 0030

fn 关键字 scanner 清单

用途：快速确认 scanner 是否已经识别 fn 关键字。

本课后应做到

  • Token 中新增 Fn
  • scanner 的 flush 闭包中已添加 "fn" => Token::Fn
  • scanner 末尾收尾逻辑中已添加 "fn" => Token::Fn
  • 新增 scanner 测试并通过

当前仍不要求

  • parser 支持函数字面量
  • AST 支持函数表达式
  • evaluator 支持函数值
  • 函数调用

最小验证样例

输入:
fn x { x + x }

目标 token:
[
  Fn,
  Ident("x"),
  LBrace,
  Ident("x"),
  Plus,
  Ident("x"),
  RBrace,
]
输入:
let add = fn(x, y) { x + y };

目标 token:
[
  Let,    Ident("add"), Equal,
  Fn,     LParen,       Ident("x"), Comma, Ident("y"), RParen,
  LBrace, Ident("x"),   Plus,       Ident("y"), RBrace, Semicolon,
]

最容易犯的错误

  • 只在 flush 闭包加 fn 分支，忘记在末尾收尾逻辑也加
  • 把 fn 误写成 Fn 去匹配
  • 一边做 scanner，一边顺手改 parser / evaluator，把课题做大
