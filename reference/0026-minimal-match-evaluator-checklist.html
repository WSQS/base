Reference 0026

最小 match evaluator 清单

用途：快速确认 evaluator 是否已经能正确执行最小 match 表达式。

本课后应做到

  • eval_expr 中已新增 Expr::Match 求值分支
  • 先求值 subject，再按 arm 顺序逐个匹配
  • 三种模式都能正确匹配：Number、Boolean、Wildcard
  • 第一个匹配的 arm 的 body 被求值并返回
  • 新增 evaluator 测试并通过 cargo test

求值规则速查

1. subject_value = eval(subject, env)
2. for arm in arms (按书写顺序):
     if pattern_matches(arm.pattern, subject_value):
         return eval(arm.body, env)
3. 全部不匹配 → 返回默认值或报错

模式匹配速查

Pattern::Number(n)   匹配 Value::Integer(v)  当 n == v
Pattern::Boolean(b)  匹配 Value::Boolean(v)  当 b == v
Pattern::Wildcard    匹配任何 Value，永远成功
类型不同 → 不匹配，跳过

最小验证样例

// 样例 1
print match 2 { 1 => 100, 2 => 200, _ => 0 };
// 输出: 200

// 样例 2
let x = 1 + 1;
print match x { 1 => 100, 2 => 200, _ => 0 };
// 输出: 200

// 样例 3
print match true { true => 1, false => 0 };
// 输出: 1

最容易犯的错误

  • 每个 arm 都重新求值 subject（应该只求一次）
  • 找到匹配后忘记 return，继续往下试别的 arm
  • Number 模式去匹配 Boolean 值时没有正确跳过
  • 顺手开始做绑定模式或 guard，把课题做大
  • 一边改 evaluator 一边又改 parser 或 AST

当前仍不要求

  • 变量绑定模式（如 x => ...）
  • guard（如 x if x > 0 => ...）
  • 穷尽性检查
  • 多语句 arm body
