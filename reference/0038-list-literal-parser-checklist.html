Reference 0038

list 字面量 parser 清单

用途：快速确认 parser / AST 是否已支持 list 字面量 [elem, ...]。

本课后应做到

  • Expr::List(Vec<Expr>) 已加入 AST
  • parser 已支持 [elem, ...]
  • 空 list [] 正确解析
  • 新增 parser / AST 测试并通过

当前仍不要求

  • evaluator 支持 list 值
  • Value::List
  • list 操作 builtin

解析逻辑速查

Token::LBracket => {
    let mut elements = Vec::new();
    let t = tokens.index(*i);
    if matches!(t, Token::RBracket) {
        *i += 1;  // 空 list
    } else {
        loop {
            elements.push(parse_expr(tokens, i));
            let t = tokens.index(*i);
            if matches!(t, Token::Comma) { *i += 1; }
            else if matches!(t, Token::RBracket) { *i += 1; break; }
            else { log!("..."); break; }
        }
    }
    Expr::List(elements)
}

最小验证样例

输入:  let x = [1, 2, 3];
目标:  Stmt::Let { expr: Expr::List([Number(1), Number(2), Number(3)]) }

输入:  [];
目标:  Expr::List([])

输入:  ["hello", 42, true]
目标:  Expr::List([String("hello"), Number(42), Boolean(true)])

最容易犯的错误

  • 空 list 没处理——看一眼 RBracket 要直接跳过
  • 解析完最后一个元素后忘记消费 RBracket
  • 一边做 parser，一边顺手去改 evaluator，把课题做大
