Reference 0035

移除 print / 表达式语句清单

用途：快速确认 scanner 已移除 print、AST/parser 已支持表达式语句。

本课后应做到

  • scanner 已移除 print 关键字
  • Stmt::Expr { expr } 已加入 AST
  • parser 已移除 Token::Print 分支
  • parser 已支持表达式语句（expr; 解析为 Stmt::Expr）
  • 测试已迁移，新增测试并通过

当前仍不要求

  • evaluator 改动
  • print builtin 注入

涉及的文件和改动

scan.rs:  删除两处 `"print" => Token::Print` （flush 闭包 + 末尾收尾）
ast.rs:   删除 `Stmt::Print`，新增 `Stmt::Expr { expr: Expr }`（删除 `Token::Print` 可选）
parse.rs: 删除 `Token::Print` 匹配分支，在 `_ =>` 兜底分支实现表达式语句

表达式语句 parser 速查

// parse() 函数的兜底分支
_ => {
    let expr = parse_expr(&tokens, &mut i);
    // parse_expr 内部遇到 ; 时会有 log，但不会移动索引
    // 检查分号，跳过它
    let t = tokens.index(i);
    if matches!(t, &Token::Semicolon) {
        i += 1;
    }
    result.stmts.push(Stmt::Expr { expr });
}

测试迁移要点

// 旧（不可用）：
print x;
print 1 + 2;
let program = parse("print x;");
assert!(matches!(..., Stmt::Print { ... }));

// 新（本课可用）：
let program = parse("x;");
assert!(matches!(..., Stmt::Expr { ... }));

// 有结果的验证改用 let 绑定：
let program = parse("let x = 1 + 2;");  // 仍然有效
let program = parse("1 + 2;");           // 表达式语句

最容易犯的错误

  • scanner 只删了 flush 闭包里的 print，忘了末尾收尾逻辑
  • parser 里表达式语句忘记处理分号
  • 测试迁移不完整——留下引用 Stmt::Print 的测试导致编译失败
  • evaluator 的 Stmt::Print 分支导致了编译错误（保留但不可达即可）
