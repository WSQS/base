Reference 0032

函数调用 parser 清单

用途：快速确认 parser / AST 是否已经支持函数调用表达式 expr(args)。

本课后应做到

  • Expr::Call { func, args } 已加入 AST
  • parser 在 primary 层末尾有 postfix 循环检查 LParen
  • expr(args) 调用能被正确解析
  • func 可以是任意表达式
  • 新增 parser / AST 测试并通过

当前仍不要求

  • evaluator 支持函数值或调用求值
  • 作用域

AST 节点速查

Expr::Call {
    func: Box<Expr>,       // 被调用的表达式
    args: Vec<Expr>,       // 实参表达式列表（可为空）
}

最小验证样例

// 样例 1：标识符调用
输入:  add(1, 2)
目标:  Expr::Call { func: Ident("add"), args: [Number(1), Number(2)] }

// 样例 2：匿名函数直接调用
输入:  (fn(x) { x + x })(3)
目标:  Expr::Call { func: Fn { ... }, args: [Number(3)] }

// 样例 3：无参调用
输入:  f()
目标:  Expr::Call { func: Ident("f"), args: [] }

// 样例 4：链式调用
输入:  f(1)(2)
目标:  Expr::Call { func: Call { func: Ident("f"), args: [Number(1)] }, args: [Number(2)] }

最容易犯的错误

  • postfix 循环只写一次检查——如果当前就是 LParen 且消费后不再继续检查，链式调用就会报错
  • 实参解析忘记处理空参数列表 ()
  • 一边做 parser，一边顺手去改 evaluator，把课题做大
