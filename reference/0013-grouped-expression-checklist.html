Reference 0013

括号表达式解析清单

用途：在 parser 第一次支持分组表达式时，快速确认你是否已经让 (expr) 作为一个整体被正确解析。

本课后应做到

  • 遇到 LParen 时不再直接报错
  • 能递归解析括号内部表达式
  • 能正确消费对应的 RParen
  • print (1 + 2); 能成功 parse 并最终输出 3

当前仍不要求

  • 完整优先级体系
  • 复杂嵌套表达式的完备支持
  • 完整错误恢复
  • 大规模 parser 重构

最小思路

parse_expr(...)
  如果当前 token 是 LParen
    跳过 LParen
    递归解析里面的表达式
    检查并消费 RParen
    返回里面的表达式

  否则
    走原来的最小表达式路径

验证样例

  • print (1 + 2); → 能 parse，并输出 3
  • let x = (1 + 2); print x; → 能 parse，并输出 3

最容易犯的 3 个错误

  • 解析完括号内部后，忘了再消费一个 RParen
  • 把括号 token 当成普通操作符处理
  • 为了做括号，一次性推倒重写整个表达式解析器
