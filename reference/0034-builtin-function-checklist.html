Reference 0034

Builtin 函数清单

用途：快速确认 evaluator 是否已经支持 builtin 函数基础设施和 add。

本课后应做到

  • Value::Builtin(fn(Vec<Value>) -> Value) 已加入
  • Expr::Call 中已处理 Value::Builtin
  • add builtin 已在环境初始化时注入
  • add(1, 2) 返回 3
  • 新增测试并通过

当前仍不要求

  • 多个 builtin（后续每个 added 都是同一条路）
  • builtin 的类型错误处理（先简单断言即可）
  • 改 scanner / parser / AST

调用分派速查

// eval_expr 中 Expr::Call 的分支
match fun {
    Value::Fn { params, body } => { /* 现有逻辑 */ }
    Value::Builtin(f) => {
        let args_vals: Vec<Value> = args.iter()
            .map(|a| eval_expr(a, env))
            .collect();
        f(args_vals)
    }
    _ => { log!("..."); Value::Integer(0) }
}

add builtin 速查

fn add_builtin(args: Vec<Value>) -> Value {
    match (&args[0], &args[1]) {
        (Value::Integer(a), Value::Integer(b)) => Value::Integer(a + b),
        _ => { log!("add expects two integers"); Value::Integer(0) }
    }
}

最小验证样例

// 样例 1：基本
输入:  let x = add(1, 2);
目标:  x = Value::Integer(3)

// 样例 2：嵌套
输入:  let a = add(add(1, 2), 3);
目标:  a = Value::Integer(6)

最容易犯的错误

  • 只在 Expr::Call 分支加了 Builtin 处理，但忘了在 eval_program 中注入 add 到初始环境
  • Builtin 的参数没有先求值就直接传进去了
  • Value::Builtin 的类型签名写错了
  • 一边做 evaluator，一边改 parser / AST，把课题做大
