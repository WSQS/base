表达式 parser 分层清单

用途：在表达式 parser 开始从“能用但混乱”整理为“能用且分层清楚”时，快速确认你是否已经把职责拆对了。

本课后应做到

  • 有清楚的表达式解析层次
  • primary 负责最基础表达式
  • mul/div 负责更紧的运算
  • add/sub 负责更松的运算

当前仍不要求

  • 完整 Pratt parser
  • 任意多优先级层级
  • 同时整理 scanner / evaluator
  • 一次性最终版 parser 架构

推荐层次

parse_expr()
  -> parse_add_sub()
      -> parse_mul_div()
          -> parse_primary()

每层职责

  • parse_primary()：数字、标识符、括号表达式
  • parse_mul_div()：在 primary 之上处理 * /
  • parse_add_sub()：在 mul/div 之上处理 + -
  • parse_expr()：表达式总入口

验证样例

  • print 1 + 2 * 3; → 输出 7
  • print (1 + 2) * 3; → 输出 9

最容易犯的 3 个错误

  • 一个函数同时处理括号、原子、优先级和递归拼接，职责仍然混在一起
  • 拆了很多函数，但每层实际职责仍然不清晰
  • 整理结构时把已有括号或最小优先级能力弄坏了
