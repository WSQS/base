v0.1 范围判断清单

用途：快速判断一个语言功能，应该放进第一版原型，还是留到以后再做。

应该进 v0.1

  • 数字字面量
  • 标识符
  • 四则运算：+ - * /
  • 括号
  • 变量定义：let name = expr;
  • 输出：print expr;

留到以后再做

  • if / while
  • 函数
  • 类型系统
  • 模块系统
  • 泛型
  • 字节码 / JIT / 优化器

判断规则

如果一个功能… 判断
能帮助你尽快打通 “源码 → token → AST → 执行” 优先进入 v0.1
会引入全新的执行模型或复杂语义 延后
没有它就无法写出最小可运行程序 优先进入 v0.1
只是为了“更强大”或“更像大语言” 延后

最小示例程序

如果你的语言已经能运行下面这样的程序，说明 v0.1 范围是合理的：

let x = 10; let y = x * 2 + 1; print y;
