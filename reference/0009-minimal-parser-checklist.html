Reference 0009

最小 parser 清单

用途：在真正写 parser 的第一步里，快速检查你是否已经开始“消费 token 并产出 AST”，而不是继续手工伪造结果。

本课后应做到

  • 根据 token 类型区分 Let 和 Print 语句入口
  • 消费最基本的 token 顺序
  • 产出对应的 Stmt 节点

当前仍不要求

  • 完整表达式优先级解析
  • 复杂错误恢复
  • 全部语句类型
  • evaluator

最小 parser 入口结构

parse_program()
  -> parse_stmt()*

parse_stmt()
  -> if next token is Let: parse_let_stmt()
  -> if next token is Print: parse_print_stmt()

当前最小验证样例

  • let x = 1; → Stmt::Let
  • print x; → Stmt::Print

最容易犯的 3 个错误

  • parse 函数仍然直接手工返回固定 AST，没有真正消费 token
  • 一开始就想把完整表达式 parser 一次写完，导致跨度过大
  • 把语句入口判断和表达式解析细节耦合得太早
