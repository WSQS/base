Reference 0010

最小二元表达式解析清单

用途：在现有最小 parser 上加入第一层 Expr::Binary 能力时，快速检查你是否真正开始构造表达式树。

本课后应做到

  • 能读取左侧原子表达式
  • 能识别一个最小二元运算符
  • 能读取右侧原子表达式
  • 能产出 Expr::Binary

当前仍不要求

  • 完整优先级体系
  • 完整括号分组解析
  • 复杂错误恢复
  • evaluator

最小思路

parse_expr()
  1. 读取左侧原子表达式
  2. 看后面是否是一个支持的二元运算符
  3. 如果是，再读取右侧原子表达式
  4. 组装成 Binary

验证样例

  • let x = 1 + 2; → 右侧是 Binary(Number(1), Plus, Number(2))
  • print x + 3; → 表达式是 Binary(Ident("x"), Plus, Number(3))

最容易犯的 3 个错误

  • 仍然只返回单个 Number / Ident，没有真正往后看操作符
  • 把语句解析和表达式解析写成一团，难以继续扩展
  • 试图��步到位做完全部优先级，导致本课跨度过大
