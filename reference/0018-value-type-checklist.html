Reference 0018

Value 类型引入清单

用途：在语言第一次把运行时结果从裸整数提升为统一值模型时，快速确认解释器接口是否已经整理到位。

本课后应做到

  • 定义了 Value 类型
  • eval_expr() 返回 Value
  • 环境保存 Value
  • 已有整数程序仍然正常工作

当前仍不要求

  • 布尔字面量
  • 比较运算
  • if / while
  • 完整类型系统

最小思路

从：
  eval_expr(...) -> i64
  env: HashMap<String, i64>

过渡到：
  eval_expr(...) -> Value
  env: HashMap<String, Value>

当前最小 Value：
  Value::Integer(i64)

验证样例

  • print 1 + 2; → 输出 3
  • let x = 5; print x; → 输出 5

最容易犯的 3 个错误

  • 只定义了 Value，但 evaluator 仍然返回裸整数
  • 环境仍然保存 i64，导致 evaluator 与环境类型不一致
  • 整数算术分支没有正确包装回 Value::Integer
