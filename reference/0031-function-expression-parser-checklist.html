Reference 0031

函数字面量 parser 清单

用途：快速确认 parser / AST 是否已经支持匿名函数字面量。

本课后应做到

  • Expr::Fn { params, body } 已加入 AST
  • parser 已支持 fn(x) { body } 字面量
  • 空参数列表和多参数列表都能正确解析
  • 新增 parser / AST 测试并通过

当前仍不要求

  • 函数调用 expr(args)
  • evaluator 支持函数值
  • 作用域
  • 多语句函数体

AST 节点速查

Expr::Fn {
    params: Vec<String>,   // 参数名列表（可为空）
    body: Box<Expr>,       // 单个表达式 body
}

最小验证样例

// 样例 1：let 绑定函数字面量
输入:  let double = fn(x) { x + x };
目标:  Stmt::Let { name: "double", expr: Expr::Fn { params: ["x"], body: Binary { ... } } }

// 样例 2：无参数
输入:  let ans = fn() { 42 };
目标:  Stmt::Let { name: "ans", expr: Expr::Fn { params: [], body: Number(42) } }

// 样例 3：多参数
输入:  fn(x, y) { x + y }
目标:  Expr::Fn { params: ["x", "y"], body: Binary { ... } }

最容易犯的错误

  • 看到 fn 后忘记先解析 ( 再解析参数列表
  • 空参数列表 fn() { ... } 没处理好
  • 解析花括号 body 时忘记调用 parse_expr
  • 一边做 parser，一边顺手去改 evaluator，把课题做大
