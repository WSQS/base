Reference 0024

match token 清单

用途：快速确认 scanner 是否已经支持未来最小 match 表达式所需的词法符号。

本课后应做到

  • scanner 已识别 match 关键字
  • scanner 已识别 _ 通配符相关 token
  • scanner 已识别 => 之类的匹配分支符号
  • 如果未来最小 match 使用花括号和 arm 分隔符，也已识别 {、}、,
  • 新增 scanner 测试并通过

当前仍不要求

  • Pattern AST
  • match parser
  • 匹配求值语义
  • 绑定模式 / guard / variant pattern

最小验证样例

let tokens = scan("match true { true => 1, false => -1, _ => 0 }");

  目标：
  scanner 能把这类未来完整 match 写法中的关键符号切成合理 token 序列。

最容易犯的错误

  • 一边加 token，一边顺手去改 parser / evaluator，把课题做大
  • 只加入 match 关键字，却漏掉 _、=> 或未来写法需要的 {、}、,
  • 测试不验证 token 顺序，只验证“有几个 token”
