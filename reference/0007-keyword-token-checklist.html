关键字识别清单

用途：在现有标识符扫描器上加入 let / print 关键字识别时，快速检查你是否维持了正确的扫描顺序。

本课后应支持

  • Let
  • Print
  • Ident(String)

当前仍不做

  • 更多关键字
  • 关键字表抽象优化
  • parser 语句解析
  • 复杂标识符规则

关键原则

1. 先把整个名字读完整
2. 再判断它是不是关键字
3. 如果是保留字，输出专门 token
4. 否则输出 Ident(...)

为什么不能过早判断

因为要遵守 maximal munch：能匹配更长的名字时，优先把它读完整。否则像 printfoo 会被错误切成 Print + Ident("foo")。

验证样例

  • "let x = 12;" → Let, Ident("x"), Equal, Number(12), Semicolon
  • "print foo;" → Print, Ident("foo"), Semicolon
  • "printfoo" → 应该是单个 Ident("printfoo")

最容易犯的 3 个错误

  • 把 let、print 仍然输出成普通 Ident
  • 过早识别关键字，错误切分更长的标识符
  • 关键字能识别，但普通名字反而失效
