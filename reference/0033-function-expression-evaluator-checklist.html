Reference 0033

函数表达式 evaluator 清单

用途：快速确认 evaluator 是否已经支持函数值和函数调用求值。

本课后应做到

  • Value::Fn { params, body } 已加入
  • eval_expr(Expr::Fn) 返回函数值
  • eval_expr(Expr::Call) 正确执行调用
  • 调用时创建局部环境（环境复制 + 参数绑定）
  • 新增 evaluator 测试并通过

当前仍不要求

  • 闭包（捕获外部变量）
  • 多语句函数体
  • 递归

调用求值步骤速查

Expr::Call { func, args } => {
    1. let f = eval_expr(func, env);  // 得到 Value::Fn
    2. let args_vals = args.iter().map(|a| eval_expr(a, env)).collect();
    3. let mut local_env = env.clone();
    4. for (param, val) in params.iter().zip(args_vals.iter()) {
           local_env.insert(param.clone(), val.clone());
       }
    5. eval_expr(body, &local_env)
}

最小验证样例

// 样例 1：基本调用
输入:  let add = fn(x, y) { x + y }; let r = add(1, 2);
目标:  r = Value::Integer(3)

// 样例 2：无参
输入:  let c = fn() { 42 }; let r = c();
目标:  r = Value::Integer(42)

// 样例 3：单参
输入:  let double = fn(x) { x + x }; let r = double(5);
目标:  r = Value::Integer(10)

最容易犯的错误

  • 函数字面量求值时直接执行了 body（应该只返回 Value::Fn）
  • 调用时没创建局部环境，参数绑定到了全局环境
  • Value::Fn 的 body 字段类型不对（应该是 Box<Expr> 以匹配 AST）
  • 一边做 evaluator，一边改 parser 或 AST，把课题做大
