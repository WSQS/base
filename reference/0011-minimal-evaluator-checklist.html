Reference 0011

最小 evaluator 清单

用途：在 Phase 0 第一次把 AST 真正执行起来时，快速确认你是否已经打通最小运行闭环。

本课后应做到

  • 能求值 Expr::Number
  • 能递归求值 Expr::Binary
  • 能执行 Stmt::Print
  • 能看到真实终端输出

当前仍不要求

  • 变量环境
  • let 的执行语义
  • 复杂错误处理
  • 完整运行时系统

最小思路

eval_expr(expr)
  Number(n) => n
  Binary(left, op, right) =>
    先递归算出 left / right
    再根据 op 做 + - * /

eval_program(program)
  遍历每条 stmt
  如果是 print expr
    先 eval_expr(expr)
    再输出结果

验证样例

  • print 1 + 2; → 输出 3
  • print 8 - 3; → 输出 5
  • print 6 / 2; → 输出 3

最容易犯的 3 个错误

  • 把 evaluator 写进 parser 里，导致“分析”和“执行”职责混乱
  • 处理 Binary 时忘了先递归求左右子表达式
  • 还没打通 print 1 + 2;，就急着做变量环境
